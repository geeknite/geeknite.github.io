## What is the QXG-10G2T-X710?

The QNAP QXG-10G2T-X710 is a PCI Express network interface card (NIC) that features two 10 Gigabit Ethernet ports. The “X710” in the model name hints at the Intel Ethernet Controller X710 series, a well-regarded workhorse in enterprise and prosumer circles. In practical terms, this card gives you two 10GBASE-T ports, which means copper RJ-45 jacks that can run over standard twisted-pair cables at up to 10 Gbps, given the right cable category (Cat6a or Cat7 typically recommended for full-fat 10G over copper at reasonable distances).

This isn’t a single-port piece of hardware that you “might” use for a future upgrade; it’s a dual-port solution designed for tasks where bandwidth matters: NAS-to-Backup, VM migration across hosts, networked storage, iSCSI targets, and heavy virtualized environments. This is the kind of card many folks pair with a 10G/10G+ + 1G copper edge switch to create a fast-lane for the data-heavy parts of their workflows while leaving the traditional 1G lanes for everything else.

However, as with all hardware that relies on drivers, ensure you have a recent driver package and keep an eye on firmware updates from Intel and QNAP. Firmware updates can fix subtle PCIe or PHY negotiation issues and can offer minor improvements in driver stability and performance.

## Performance expectations and bench talk

Let’s talk numbers without pretending we’re bench gods. In a properly configured environment (two parallel 10G links on a capable NAS switch, good copper cabling, and workloads that actually saturate the links), you should expect near-theoretical 10 Gbps speeds in sustained transfers. In practice, you’ll often see a little overhead from protocol processing, encryption offloads (if enabled), and the virtualization stack if you’re using iSCSI or NAS-based VM storage migrations. With jumbo frames enabled (MTU 9000), you’ll often see more efficient throughput for large file transfers because the CPU overhead per data unit drops.

A practical scenario: a sizable 4–8 TB backup over SMB to a NAS fronted by the QXG-10G2T-X710 can sustain a healthy fraction of 10 Gbps, often in the 6–9 Gbps range depending on the workload mix, the VM count, and whether you’re encrypting traffic on the fly. If you’re doing multiple concurrent streams—backups, VM migrations, and user downloads—the two 10G ports help keep all lanes busy without starving any single workflow.

Remember: the NIC doesn’t create bandwidth out of thin air. Your drives, RAID configuration, and network switch all matter. If your NAS is pushing to a single 10G link and you’re trying to saturate both ports simultaneously, you’ll get better results from a balanced network design: distribute traffic across VLANs or networks, and use LACP to aggregate where you truly need it.

## Setting up on a QNAP NAS: step-by-step guidance

1) Power down the NAS and install the QXG-10G2T-X710 in an available PCIe slot. If you’re working in a small form factor chassis, grab the low-profile bracket.
2) Power on and enter the QNAP reboot process. Open the QTS/QuTS hero interface and navigate to the Network Control panel where you configure NICs.
3) The two 10G ports should appear as eth1 and eth2 (or similar naming). Enable them and assign IPs or configure them to obtain IPs via DHCP if your network design requires it.
4) Configure link aggregation (LACP) if your switch supports it: both ports can be part of a single logical 10G link for higher throughput or failover. Alternatively, keep them separate for different services (e.g., one port for SMB/DFS, another for iSCSI).
5) If you plan to use VLANs, create VLAN interfaces off the 10G NICs to segment traffic. This is especially useful in mixed environments where you separate management, storage, and VM traffic.
6) For large file transfers, consider enabling jumbo frames (MTU 9000) on both the NICs and the corresponding switch ports. This can improve throughput for large payloads, but you must ensure the entire path supports jumbo frames.
7) Test the configuration with a large file transfer between NAS clients and your host or another NAS. Verify both read and write performance and ensure there is no packet loss or unusual retransmissions.
8) If you use virtualization, configure VMs to use a dedicated NIC or a dedicated virtual network for storage traffic to minimize cross-TCP competition.

If you’re a fan of “set it and forget it,” you’ll be happy with the X710’s stable performance and the fact that Intel’s driver support makes updates smoother without constant reconfiguration. If you enjoy fine-tuning, you’ll appreciate the ability to carve up the network with VLANs and multiple paths.

## Compatibility caveats and troubleshooting tips
- Ensure your NAS supports PCIe upgrades and that you’re using a compatible slot (x4 or higher, with enough bandwidth for two 10G ports).
- If you don’t see the NIC after installation, reseat the card, reseat the brackets, and verify the PCIe slot is enabled in the motherboard’s BIOS (some consumer boards default to disabled PCIe lanes in certain configurations).
- Double-check cabling: Cat6a is the minimum to reliably reach 10G over copper; Cat7 offers more headroom for longer runs. Duplex settings should be full duplex for best results.
- If jumbo frames are enabled but you experience issues, revert to MTU 1500 until all devices on the path are aligned.
- For virtualization-heavy workloads, keep an eye on CPU usage on the NAS host. Offloading features on the X710 are helpful, but heavy VM networking can still tax the host CPU if you’re not careful.

### Alternative paths to consider
If you’re not ready to jump into a dual 10G copper solution, consider a single-port 10G card, or a 2.5G/5G approach as a stepping stone. For some budgets, a single 10G NIC with a capable switch and proper VLAN segmentation yields much of the day-to-day benefit without the extra complexity of dual-path configurations. But if you know you’ll need that extra headroom later, starting with a dual-port 10G is a future-proof move.
