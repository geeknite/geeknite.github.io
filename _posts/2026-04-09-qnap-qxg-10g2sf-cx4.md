## Specs and design choices

The QXG-10G2SF-CX4 is built around two SFP+ ports, which means 10Gbps per port, with a total theoretical bandwidth of 20Gbps. In real world terms, you’ll be lucky to approach the full 20Gbps across both directions unless you’ve got a pair of top-tier NAS devices and a switch that doesn’t choke on line rate. Still, it’s a substantial improvement over a single gigabit NIC and is a nice match for modern NAS devices that can saturate 10GbE links.

- Form factor: PCIe 3.0 x4
- Ports: 2 x SFP+ (not including optics)
- Bracket: Low-profile included
- Driver landscape: Linux, Windows, and NAS environments typically supported; check your distribution for driver availability
- Cables/optics: Requires SFP+ transceivers or DAC cables (not included)

Notice that the card leaves you to pick your own SFP+ optics. That’s not a failure; that’s a feature. You’re not locked into a single vendor or cable standard. You can fine-tune the network for your distance and budget, choosing SR (short range) for fiber runs, LR for longer copper-free runs, or a DAC (direct attach copper) for short hops across a rack.

## Setup, installation, and caveats

- Slot into PCIe x4: It’s the right bandwidth for two 10GbE lanes; you’ll probably want to install into a free PCIe x4 or x8 slot if possible to avoid any potential bottleneck on busy systems.
- Bracket: The CX4 includes a low-profile bracket, a blessing for small form factor builds and 2U NAS boxes. If you're mounting in a big chassis with generous clearance, you can leave it as is or swap for a taller bracket if necessary.
- Optics: Acquire SFP+ transceivers or DAC cables compatible with your network gear. If your switch uses SR optics, you’ll want SR transceivers; if it uses LR optics, LR transceivers, and so on.
- Driver installation: In Linux, the kernel usually detects the NIC and loads the drivers automatically. Windows users may need to install driver packages from QNAP or the NIC vendor. On NAS OS, check the firmware notes to ensure compatibility.

Recommended steps:
1) Power down, install the card in a clean PCIe slot, and secure with the screw.
2) Attach the appropriate SFP+ modules or DAC cables.
3) Boot, confirm the NIC shows up in your OS’ network interface list, and assign IPs or configure bonding if you want to aggregate links.
4) If you’re using Linux, test with iperf3 between machines to verify throughput; on NAS, test with file transfers to gauge real-world performance.

Common gotchas:
- Not all NAS firmware builds include drivers for every PCIe NIC by default; ensure your firmware is up to date.
- If you’re using a non-QNAP NAS, verify kernel module compatibility before purchase.
- When using an LACP bond, ensure the switch port group is configured for LACP and that you’re using compatible switch hardware.

### Quick setup checklist
- Choose appropriate SFP+ modules for your distance and fiber type (SR, LR, or DAC).
- Install the card in a free PCIe slot and secure with the screw.
- Attach the cables, boot the system, and install the necessary drivers if not already present.
- Configure your link aggregation (if desired) on both sides (NAS and switch/router) using LACP or static multi-link depending on your hardware.
- Validate throughput with a reliable test like iperf3 and a large block size.

There’s nothing inherently loud about the QXG-10G2SF-CX4. It’s a modest heat source, no louder than a typical PCIe NVMe drive in a case with semi-decent airflow.
